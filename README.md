## Quick Start

### 1. Install dependencies

```bash
# GPU — college machines (check CUDA version with nvidia-smi first)
pip install -r requirements-gpu.txt --break-system-packages

# Also install fonts for Telegraph synthetic training data:
sudo apt-get install -y fonts-liberation fonts-freefont-ttf

# CPU only (works everywhere, slower training)
pip install -r requirements.txt
```

### 2. Train the model

```bash
# Always run from project ROOT
python backend/train.py
```

This trains a CNN specifically tuned for **The Telegraph newspaper** sudoku
font using Liberation Sans / FreeSans synthetic data mixed with augmented
inverted MNIST. Best checkpoint saved to `backend/model_weights/digit_cnn.pth`.

Training time: ~25–40 min on GPU · ~4–6 h on CPU

### 3. Run the app

```bash
# Always run from project ROOT
python app.py
# → http://127.0.0.1:5000
```

## Model Architecture

```
Input (1, 28, 28) — BLACK digit on WHITE background
│
├─ Block 1: Conv(1→32)×2 + BN + ReLU + MaxPool + Dropout2d
│  └─ SE Attention (channel squeeze-excitation, r=8)
│
├─ Block 2: Conv(32→64)×2 + BN + ReLU + MaxPool + Dropout2d
│  └─ SE Attention
│
├─ Block 3: Conv(64→128)×2 + BN + ReLU + MaxPool + Dropout2d
│  └─ SE Attention
│
└─ Classifier: Flatten → 512 → ReLU → Dropout(0.5)
                       → 256 → ReLU → Dropout(0.3)
                       → 10 logits
```

**SE Attention** suppresses grid-line noise feature maps and amplifies
digit-stroke features — critical for thin-stroke newspaper printing.

## Training Dataset

| Augmentation         | Weight | Purpose |
|---|---|---|
| TelegraphFont (Liberation/FreeSans) | ×4 | **Primary** — matches actual font |
| InvertedMNIST base  | ×1 | General digit variety |
| HardPairTransform   | ×3 | 1↔7, 2↔7, 3↔8, 6↔9 confusion pairs |
| PerspectiveTransform| ×2 | Skewed / angled camera shots |
| GeometricTransform  | ×1 | Rotation / scale / translate |
| NoiseTransform      | ×1 | Paper grain + lighting variation |

**Normalisation:** mean=0.8693, std=0.3081 (inverted MNIST, must match in both
`train.py` and `ocr.py`)

## OCR Pipeline

```
Image bytes
 │
 1. Decode + resize (800–2000px)
 2. Grid detection — 5-strategy cascade:
    ① Adaptive Gaussian threshold + largest quad
    ② Adaptive Mean threshold + largest quad
    ③ Otsu threshold + largest quad
    ④ Morphological line detection
    ⑤ Hough line intersection (extreme skew fallback)
    → Best-scored quad wins
 3. Perspective warp → 450×450 px
 4. Extract 81 cells (50×50 px, 5px margin)
 │
 For each cell:
   Gate 1  OpenCV contour check → blank? skip CNN
   Layer 1 DigitCNN inference
           conf ≥ 0.72 → accept directly
           conf ∈ [0.28, 0.72) → accept, flag uncertain
           conf < 0.28 → treat as blank
   Layer 2 Sudoku constraint correction
           Pass 1: already solvable? return
           Pass 2: single substitution (conflict cells first)
           Pass 3: double substitution (conflict pairs)
           Pass 4: blank-cell recovery
           Pass 5: nuclear — zero lowest-confidence cells until solvable
```

## Key Design Principles

- **Wrong predictions never acceptable** — constraint correction Pass 5
  (nuclear option) guarantees a solvable board by zeroing uncertain cells
  rather than leaving wrong digits that block the solver.
- **Same font always** — TelegraphFont synthetic dataset targets Liberation
  Sans / FreeSans which closely matches The Telegraph's actual sudoku font.
- **Severe skew handled** — 5-strategy grid detection cascade; Hough line
  intersection as final fallback handles near-perspective-flat shots.
- **No EasyOCR** — single-digit crops are outside EasyOCR's training
  distribution; it hallucinates "8" from grid-line artefacts.
- **Normalisation discipline** — mean=0.8693 (inverted MNIST) used
  consistently across train.py and ocr.py; any mismatch breaks inference.
